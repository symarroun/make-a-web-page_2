HTML

HTML stands for HyperText Markup Language. HTML is primarily used for creating web documents which make the world wide web of the internet. The content of the web page is written within the HTML code. They syling can be added within the same code too. However, it is preferred to style the page by linking a CSS code to the HTML in order to avoid code repetition which will be explained later.

HTML Elements and Tags

What goes between < > are called tags. HTML elements are contents surrounded by tags telling the computer what to do with the contents. Some elements require attribues to be added within a tag. The attribute is written in the opening tag with its appropriate chosen value which usually vary.

Keep in mind that each element has an opening tag and a closing tag but there are some exceptions. Some elements like the image <img> tag is an element without a closing tag and is referred to as an element with a void tag.

Elements are divided in two main categories:
  • Inline Elements: span, a, br, img, strong
  • Block elements: div, p, form

Document Code Structrue ( Div and Span )

Before getting into what a structure means, it is important to be aware that lines of codes do have relationships amongst each other. e.g. a child, a parent, or a sibling. Structuring a document with indentation within the code does not tell HTML anything since the whitespace is ignored. Thus, the tag <div> </div> is used for the purpose of turning the page into boxes (boxifying).
Div tags are considered as container units which divide elements in the HTML code document into sections. Programmers use this element to group HTML elements which helps applying CSS styles and avoid repitition.

Typical HTML Document Structure

<! Doctype HTML>
<html>
<head>
<title>TheTitle</title>
</head>
<body>
The Content along with its elements and tags
</body>
</html>

CSS

CSS stands for Cascading Style Sheet. It is useful as a tool for how the content is being presented on the web page.

HTML "Classes" and Avoiding Repetition

If you wondered why do we use CSS for styling while it is possible to do so in HTML. Well, there are several benefits for doing so.

  1. Most importantly is the idea of avoiding repetition within the code.
  2. Focusig on the content within the HTML code helps making the presentation better later on.
  3. Linking multiple .css files to one HTML comes in handy when working on a full website. This also helps to simply apply different .css for the same .html depending on which device (i.e. laptop, phone, or tablet) the web page is being viewed from.
